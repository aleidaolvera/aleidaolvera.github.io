Basics and Terminology

CSS stands for Cascading Style Sheets Styles. CSS defines how to display HTML elements. CSS was added to HTML 4.0 to solve a problem with External Style Sheets and in turn can saved a lot of work. HTML was never intended to contain tags for formatting a document. To solve this problem, the World Wide Web Consortium (W3C) created CSS. In HTML 4.0, all formatting could be removed from the HTML document, and stored in a separate CSS file. This file is achieved by adding .css to the end of your file name, just like in HTML! All browsers fully support CSS presently.

Common CSS Terms

  • Selectors- Selectors determine exactly which element, or elements, the corresponding styles will be applied to. Selectors can include any IDs, classes, types, and other attributes, it just depends on how specific you wish to be. Selectors can be identified as everything that comes before the first curly brace, {.
  • Properties- A property determines the style that will be added to a designated element. Properties can be recognized as the text coming right before a colon. There are many properties you can use, and new ones are always being added.
  • Values- Values determine the behavior of a property. Values can be recognized as the text in the middle of the colon and semicolon.

Just like in HTML there is a basic structure you need to follow in order for your CSS to be read. You must have this under your header, or a link to an external CSS file. An external CSS file is just a document separate from your HTML that is just dedicated to CSS.
